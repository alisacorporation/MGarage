# MGarage — журнал ремонта автомобилей

Личное офлайн-приложение для учёта ремонтов, доходов и клиентов автомастерской.
Работает полностью на устройстве: без регистрации, без сервера, без облака,
без Firebase/Supabase, без единого сетевого запроса к сторонним API. Все данные
хранятся локально в IndexedDB браузера.

## Запуск

**Вариант 1 — напрямую:**
Откройте `index.html` в браузере (Chrome/Edge/Safari).

**Вариант 2 — как PWA на Android:**
1. Разместите папку `MGarage/` на любом статическом хостинге (GitHub Pages,
   Netlify, обычный веб-сервер) — HTTPS обязателен для service worker и
   установки PWA.
2. Откройте сайт в Chrome на Android.
3. Нажмите «Установить» (кнопка в шапке приложения) или используйте меню
   браузера → «Добавить на главный экран».
4. Приложение будет работать полностью офлайн после первого запуска.

**Вариант 3 — локальный статический сервер (для разработки):**
```bash
cd MGarage
python3 -m http.server 8080
```
Откройте `http://localhost:8080`.

> Открытие `index.html` напрямую через `file://` работает для интерфейса и
> IndexedDB, но service worker (офлайн-кэш и установка PWA) требует HTTP(S)
> — это ограничение всех браузеров, не приложения.

## Возможности

- **Главная** — доход за сегодня/неделю/месяц, общая прибыль, число машин, последние работы.
- **Новая работа** — полная карточка ремонта: авто, VIN, госномер, клиент,
  стоимость/расходы с автоматическим расчётом прибыли, чек-листы
  «Замена»/«Диагностика», фото до/после.
- **История** — поиск в реальном времени по марке, модели, VIN, госномеру,
  клиенту; фильтр по маркам; редактирование и удаление записей.
- **Статистика** — доход за день/неделю/месяц/год, средний чек, самая
  прибыльная и самая частая марка, график дохода за 6 месяцев и диаграмма
  распределения по маркам (Canvas, без сторонних библиотек).
- **Резервная копия** — полный экспорт базы в JSON-файл и полное
  восстановление из файла. Работает без сети.

## Технологии

HTML5, CSS3 (собственные стили в духе BMW iDrive/M, без Bootstrap/Tailwind
CDN — чтобы приложение оставалось на 100% офлайн без зависимости от сети при
первом запуске), JavaScript ES2023 (без TypeScript, без сборщиков),
IndexedDB, Canvas для графиков, Web App Manifest + Service Worker для PWA.

## Структура проекта

```
MGarage/
  index.html        — разметка приложения
  style.css          — стили (BMW M тема)
  db.js              — слой IndexedDB (cars, jobs, photos, settings)
  ui.js              — рендеринг страниц, форм, модалок
  app.js             — инициализация, роутинг, install prompt, service worker
  charts.js          — графики на Canvas
  search.js          — поиск и фильтрация истории
  export.js          — экспорт JSON
  import.js          — импорт JSON
  manifest.json       — PWA-манифест
  service-worker.js   — офлайн-кэш
  icons/              — иконки 192/512, включая maskable
  assets/             — резерв для доп. ресурсов
```

## Хранение данных

Все данные (машины, работы, фотографии, настройки) хранятся в базе
IndexedDB `mgarage_db` внутри браузера/WebView на устройстве пользователя.
Приложение не отправляет данные никуда. Резервное копирование выполняется
вручную через раздел «Резервная копия» → экспорт/импорт JSON-файла.

## Заметка о Tailwind

Бриф допускал использование Tailwind. Чтобы гарантировать полностью
офлайн-первый запуск без единого сетевого запроса (включая Tailwind Play
CDN, который сам по себе требует сети), интерфейс реализован на чистом CSS
в собственной дизайн-системе, повторяющей эстетику BMW M/iDrive
(тёмный фон, стеклянные карточки, фирменная трёхцветная полоса
navy/blue/red). Это осознанный выбор в пользу требования «полностью
офлайн», а не отказ от Tailwind как такового.
